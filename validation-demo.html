Quick Start

Open docs-validation.js and find the VALIDATION_SCHEME object. Each field has simple properties you can modify.

Example 1: Basic Text Field

'invoice_no': {
    required: true,           // ‚úÖ Make required
    type: 'text',
    minLength: 1,            // Minimum characters
    maxLength: 30,           // Maximum characters
    pattern: /^[A-Z0-9\-\/]+$/,  // Only letters, numbers, hyphens, slashes
    errorMessage: 'Invoice number is required and must contain only letters, numbers, hyphens, and slashes'
}

Example 2: Number Field

'invoice_value': {
    required: false,         // ‚ùå Not required
    type: 'number',
    min: 0,                  // Minimum value
    max: 999999999,          // Maximum value
    errorMessage: 'Invoice value must be a positive number'
}

Example 3: Date Field

'invoice_date': {
    required: true,
    type: 'date',
    minDate: '2020-01-01',   // Earliest allowed date
    maxDate: null,           // null = no limit
    errorMessage: 'Invoice date is required and must be after 2020'
}

Example 4: Select Field

'currency': {
    required: false,
    type: 'select',
    options: ['USD', 'EUR', 'GBP', 'AUD', 'CAD', 'AED', 'INR'],
    errorMessage: 'Please select valid currency'
}

üéØ Common Editing Tasks

Add a New Field

To add validation for a new field, simply add it to the VALIDATION_SCHEME object:

'new_field_name': {
    required: true,
    type: 'text',
    minLength: 5,
    maxLength: 100,
    errorMessage: 'This field is required (5-100 characters)'
}

Make a Field Required/Optional

Simply change the required property:

// Make required
'field_name': { required: true, ... }

// Make optional  
'field_name': { required: false, ... }

Change Validation Rules

Modify any validation property:

// Change length limits
minLength: 10,    // was 5
maxLength: 200,   // was 100

// Change number ranges  
min: 1,          // was 0
max: 50000,      // was 999999999

// Update error message
errorMessage: 'New custom error message here'

Add Pattern Validation

Use regular expressions for custom patterns:

// Email pattern
pattern: /^[^\s@]+@[^\s@]+\.[^\s@]+$/

// Phone number pattern (Indian)
pattern: /^[6-9]\d{9}$/

// Alphanumeric only
pattern: /^[A-Za-z0-9]+$/
